# Swahili GPT

A character-level language model trained on **Swahili news articles** — built to explore how a GPT-style Transformer performs on an East African language with a rich morphological structure.

Two models are implemented and compared:

| Model | Architecture | Parameters |
|-------|-------------|-----------|
| `bigram.py` | Token + position embeddings → linear head | ~200K |
| `gpt.py`    | 6-layer decoder-only Transformer (nanoGPT-style) | ~10M |

Both are trained character-level, meaning the model learns Swahili spelling, morphology, and partial syntax from scratch with no prior language knowledge.

---

## Why Swahili?

Swahili (Kiswahili) is spoken by 100–150 million people across East Africa and is the official language of Kenya, Tanzania, Uganda, and the African Union. Despite this, it remains severely under-represented in NLP research compared to European languages.

Training a generative model on Swahili demonstrates:
- How character-level models handle **agglutinative morphology** (Swahili words carry tense, subject, object agreement as prefixes/suffixes)
- The difference in **vocabulary complexity** vs. English — Swahili has a smaller character set but denser morphological structure per token
- A baseline for future work on **low-resource African language modeling**

## Dataset

**Swahili News** — [HuggingFace `swahili_news`](https://huggingface.co/datasets/swahili_news)

- 31,000+ news articles from Tanzanian online platforms
- Categories: local, international, business, health, sports, entertainment
- License: CC BY 4.0

The dataset is downloaded and prepared automatically by `data/prepare.py`.

## Quickstart

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Download the corpus

```bash
python data/prepare.py
```

This saves ~31,000 Swahili articles (~40MB) to `data/swahili.txt`.

### 3. Train the bigram baseline

```bash
python bigram.py
```

Trains in a few minutes on CPU. Outputs `bigram_loss.png`.

### 4. Train the GPT model

```bash
python gpt.py
```

Trains for 5,000 steps. Saves checkpoint to `gpt_swahili.pt` and outputs `gpt_loss.png`.

> **GPU recommended** for the GPT model. On CPU expect ~2–3 hours for 5,000 steps.
> MPS (Apple Silicon) and CUDA are auto-detected.

### 5. Generate interactively

```bash
# Interactive prompt loop
python generate.py

# Single-shot with a prompt
python generate.py --prompt "Rais wa Tanzania alisema" --tokens 400

# Adjust creativity
python generate.py --temperature 0.8 --tokens 600
```

## Architecture Notes

### Bigram (baseline)

```
Input tokens
    → Token Embedding  (vocab_size × 32)
    → Position Embedding (block_size × 32)
    → Linear Head  (32 → vocab_size)
    → Cross-entropy loss
```

No attention. Each character predicts the next purely from its embedding. A useful lower-bound baseline.

### GPT (full model)

```
Input tokens
    → Token Embedding  (vocab_size × 384)
    → Position Embedding (256 × 384)
    → 6 × Transformer Block:
          LayerNorm
          Multi-Head Causal Self-Attention  (6 heads × 64 dims)
          Residual connection
          LayerNorm
          Feed-Forward Network (384 → 1536 → 384, ReLU)
          Residual connection
    → LayerNorm
    → Linear Head  (384 → vocab_size)
    → Cross-entropy loss
```

Trained with AdamW (lr=3e-4), dropout=0.2, context window of 256 characters.
